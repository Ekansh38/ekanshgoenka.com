byte-space
Go · active · Unix sockets · protocol implementation

A simulator of the 1980s internet. Every protocol implemented from scratch over raw Unix sockets — no net/http, no standard library shortcuts. The goal is fidelity: a running system that behaves the way the early internet actually behaved.

What it implements

HTTP/1.0, SMTP, DNS resolver, and Telnet — each as a standalone server and client pair. Protocols communicate over Unix domain sockets, with a small routing layer that connects them into a single simulated network. You can send mail, resolve names, and fetch pages, all within the same process.

Design decisions

Everything is written in Go, which gives clean concurrency primitives without hiding the socket layer. Each protocol lives in its own package with a strict interface boundary. No external dependencies. The DNS implementation uses a hand-rolled recursive resolver with a local root zone.

Status

HTTP and DNS are stable. SMTP is functional but missing AUTH. Telnet supports basic VT100 negotiation. Currently working on a unified shell that lets you interact with all four protocols from a single terminal interface.
